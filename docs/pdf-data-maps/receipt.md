# Receipt PDF — data map

**View model:** `App\Services\Pdf\ViewModels\ReceiptPdfViewModel`
**Source data:** `Receipt::findWithDetails($id)` (joins payments, invoices, customers)
**Recalculated before render:** the invoice row is the source of `total / amount_paid / balance_due`. Those values are kept in sync by `Invoice::applyPayment` whenever a payment is recorded; the view model does not silently fix mismatches.

## What this PDF intentionally does NOT show

* **QR code for Google review** — present in the mockups, but the company review-link config and QR generation are not implemented yet.
* **Captured signature image** — `customer_approvals` carries the signature file id, but there is no PDF image XObject support in the hand-rolled renderer. The signature line is left blank for offline pen-on-paper, and the digital signature is stored separately as a `file_attachment`.
