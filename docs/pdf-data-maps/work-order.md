# Work Order PDF — data map

**View model:** `App\Services\Pdf\ViewModels\WorkOrderPdfViewModel`
**Source data:** `WorkOrder::findWithDetails($id)` (joins service_requests, estimates, customers, vehicles, locations)
**Recalculated before render:** no money is recomputed on this PDF — the work order doesn't carry line items. The `estimate_total` shown comes from the joined `estimates.total`, which the Estimate flow keeps current via `Estimate::recalculate()`.

## What this PDF intentionally does NOT show

* **Money line items / totals.** The WO defers itemisation to the linked Estimate (before work) and the eventual Invoice (after work). Showing recomputed amounts here would invite the two to disagree.
* **Customer signature.** Per company doc principle 1, the customer signs only the Waiver (before work) and the Receipt (at completion). The WO is internal.
* **Assigned user / technician name.** `work_orders.assigned_user_id` exists in the schema but the column is not yet populated by any controller path; surface it only once the assign flow is wired.
