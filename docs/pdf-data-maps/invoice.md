# Invoice PDF — data map

**View model:** `App\Services\Pdf\ViewModels\InvoicePdfViewModel`
**Source data:** `Invoice::findWithDetails($id)` + `InvoiceLineItem::forInvoice($id)`
**Recalculated before render:** yes — `DocumentController` calls `Invoice::recalculate($id)` before constructing the view model.

## What this PDF intentionally does NOT show

* **Signature line** — Invoice doesn't require a customer signature (Receipt does, per company doc principle 1).
* **Inline payment record** — Payments live on the separate Receipt PDF; the invoice only shows the aggregate `amount_paid` and `balance_due` from the invoice row.
* **Mockup discount column** — there is no discount field on `invoice_line_items` today. Adding one needs a migration first.
