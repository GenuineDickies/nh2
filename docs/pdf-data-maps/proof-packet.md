# Proof Packet PDF — data map

**View model:** `App\Services\Pdf\ViewModels\ProofPacketPdfViewModel`
**Source data:** `ServiceRequest::proofPacket($id)` — a composite that joins service_request → estimate → approval → work_order → service_completion_report → invoice → payments → receipts → ledger_entries → attachments → generated_documents → timeline.
**Recalculated before render:** the packet returns whatever is currently stored — by design, because the packet is an audit snapshot rather than a billable document. Individual money totals come from already-recalculated rows.

## What this PDF intentionally does NOT show

* **Audit log timeline** — the packet exposes `timeline[]` already, but rendering 20+ rows of audit entries inflates the PDF without aiding the reader. The audit trail is accessible separately in the app.
* **Embedded photos / signature images** — would require PDF image XObject support; the hand-rolled renderer doesn't have it.
