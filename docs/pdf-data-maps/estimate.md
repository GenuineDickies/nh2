# Estimate PDF — data map

**View model:** `App\Services\Pdf\ViewModels\EstimatePdfViewModel`
**Source data:** `Estimate::findWithDetails($id)` + `EstimateLineItem::forEstimate($id)`
**Recalculated before render:** yes — `DocumentController` calls `Estimate::recalculate($id)` before constructing the view model.

## What this PDF intentionally does NOT show

* **Signature line** — Estimates are quotes, not contracts. Approval is captured separately on `customer_approvals` (via the tokenised public-portal page at `/p/estimate/{token}`, phone confirmation, or on-site signature). The system does NOT send the portal link automatically — the operator shares the URL manually. SMS / email delivery would require the not-yet-implemented Telnyx + email integrations called out in [TODO.md](TODO.md).
* **Customer service address** — present in the mockups but lives on the `service_request → location` join, not on the estimate row. Add only when the join is wired.
* **Logo image** — `PdfDocument::masthead` draws an outlined initials placeholder. A real raster logo needs a PDF XObject (out of scope for the hand-rolled writer).
